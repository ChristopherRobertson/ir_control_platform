# V1 Workflow Backend Map

This file replaces the older experiment-page wiring ledger.

## Active Pages
- Session -> UI route `/session`
- Setup -> UI route `/setup`
- Results -> UI route `/results`

## Backend Ownership
- Session save and run header save are persisted by `data-pipeline`.
- Setup validation and run-button gating are owned by `experiment-engine`.
- Run starts from Setup and freezes a `RunSettingsSnapshot`.
- Raw and processed run records are persisted by `data-pipeline`.
- Results reads saved records and uses `processing` transforms through the platform runtime.
- Exports are generated by `reports` from persisted artifacts.

## Removed From Active Map
Device-first routes, broad device pages, direct UI-to-device actions, standalone readiness pages, live plotting, and wavelength sweep controls are not part of v1.
